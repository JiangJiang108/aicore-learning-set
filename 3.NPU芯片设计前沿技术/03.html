3. NPU核心架构：脉动阵列与数据流设计

各位同学，今天我们来聊聊NPU最核心的架构——脉动阵列。说实话，这个结构我研究了快十年，每次看都觉得它有种数学上的美感。咱们从原理开始，一步步拆解。

3.1 脉动阵列的基本原理

脉动阵列（Systolic Array）这个名字，其实挺形象的。你想象一下心脏泵血，血液有节奏地流向全身。在芯片里，数据就像血液一样，在计算单元（PE）之间有节奏地流动。

我当年第一次接触这个概念，是在读Kung教授1982年的论文。那时候觉得：这玩意儿真聪明！为什么？因为它解决了芯片设计里一个老大难问题——数据搬运比计算还耗能。

核心思想：让数据在PE之间"流"起来，每个PE只做简单的乘加运算，数据复用率极高。说白了，就是用空间换时间，用局部性换效率。

你看这个图，数据从左边流进来，权重从上边流下来。每个PE只跟邻居通信。这种局部性，正是硬件设计者梦寐以求的。

3.2 三种数据流架构

数据流架构，说白了就是决定"谁不动、谁在动"。我参与过三个NPU项目，每个项目选的数据流都不一样。咱们一个一个说。

3.2.1 Weight Stationary（权值固定）

核心思路：权重加载到PE后就不动了，输入特征图在阵列里流动，部分和在PE间累加传递。

我个人习惯在卷积层用这种架构。为什么？因为卷积的权重复用率极高。你想想看，一个3×3的卷积核，要在整个特征图上滑动，权重被反复使用。让权重待在PE里不动，能省掉大量搬运功耗。

我的经验：在7nm工艺下，一次DRAM访问的能耗大约是SRAM的100倍。Weight Stationary能减少权重搬运，对能效提升非常明显。我曾在某个项目里，靠这个设计把能效比提升了3倍。

3.2.2 Output Stationary（输出固定）

核心思路：输出部分和在PE里累加，输入特征图和权重都在阵列里流动。

这种架构适合全连接层。我记得有一次做BERT的NPU加速，全连接层的计算量占了大头。用Output Stationary，每个PE负责一个输出神经元的部分和累加，最后再汇总。这样能减少写回存储器的次数。

不过要注意，Output Stationary对PE的本地存储要求更高。每个PE需要更大的累加器，不然精度会丢。

3.2.3 Systolic Dataflow（脉动数据流）

这是Google TPU用的方案。输入特征图和权重都在流动，但方向不同。就像我前面画的图那样，一个水平流，一个垂直流。

这种架构的优点是平衡性好。没有哪个数据是"固定"的，所以阵列的利用率可以做到很高。但代价是控制逻辑复杂一些。

特性 Weight Stationary Output Stationary Systolic Dataflow
适用场景 卷积层 全连接层 通用矩阵乘
数据复用 权重复用高 输出复用高 均衡
PE存储需求 中等（存权重） 较大（存累加器） 中等
控制复杂度 低 中 高
能效比 高（卷积场景） 高（FC场景） 中高

3.3 片上存储层次设计

存储层次，这是NPU设计里最容易被低估的部分。我曾经吃过亏——第一版芯片流片回来，算力够，但带宽不够，数据喂不饱计算单元。那叫一个惨。

一个典型的NPU存储层次是这样的：

  1. 全局缓冲区（Global Buffer）：通常几百KB到几MB，用SRAM实现。这是离计算阵列最近的"大仓库"。
  2. 局部寄存器（Local Register）：每个PE内部的小存储，几十到几百字节。存权重或部分和。
  3. 累加器（Accumulator）：专门存累加结果的寄存器，精度要高，我一般用32位浮点或16位。

避坑指南：我曾经在一个项目里，把全局缓冲区设计得太大，结果面积爆炸，时序跑不过。后来学乖了——先算带宽需求，再定缓冲区大小。公式很简单：Buffer Size = 计算阵列吞吐量 × 数据复用周期数。

这里给个参考配置：

// 一个典型的NPU存储配置示例
// 假设：128×128的脉动阵列，INT8精度

#define PE_ROWS 128
#define PE_COLS 128
#define DATA_WIDTH 8  // bits

// 全局缓冲区：2MB SRAM
// 可以存 2MB / (128×128×1B) ≈ 128 层特征图
#define GLOBAL_BUFFER_SIZE (2 * 1024 * 1024)  // bytes

// 每个PE的局部存储：256字节
// 可以存 256 / 1 = 256 个权重值
#define PE_LOCAL_MEM_SIZE 256  // bytes

// 累加器：32位精度，每个PE一个
#define ACCUMULATOR_WIDTH 32  // bits

3.4 计算单元（PE）设计

PE是脉动阵列的"细胞"。设计得好不好，直接决定芯片的算力和能效。

一个典型的PE包含：

  • 乘法器：INT8或FP16，看应用场景
  • 加法器：支持累加操作
  • 本地寄存器：存权重或中间结果
  • 数据通路：跟邻居通信的接口

我建议PE设计时注意三点：

  1. 位宽选择：INT8对大多数推理任务够用，但训练场景需要FP16甚至FP32。别盲目追求高位宽，面积和功耗会翻倍。
  2. 流水线深度：我一般用2-3级流水线。太深了面积大，太浅了频率上不去。
  3. 数据复用：PE内部的数据通路要设计好，尽量减少不必要的寄存器读写。

核心设计原则：PE的利用率要达到80%以上才算合格。如果低于这个数，说明你的数据流或存储层次有问题。我见过太多设计，PE阵列做得很大，但实际利用率不到50%——那还不如做小一点，省下来的面积做缓存。

好了，关于脉动阵列的核心架构，咱们就聊到这儿。这些内容是我多年实践的心得，希望能帮到你。记住，架构设计没有银弹，关键是根据应用场景做取舍。
