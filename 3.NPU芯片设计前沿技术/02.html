神经网络基础：从感知机到Transformer

各位同学，今天我们来聊聊神经网络的基础。说实话，这部分内容我讲了不下几十遍，但每次备课都会有新感悟。神经网络这东西，你越深入越觉得它有意思。

一、感知机与多层感知机

感知机是最简单的神经网络单元。1957年Frank Rosenblatt提出它的时候，整个学界都沸腾了——机器居然能"学习"了！

一个感知机说白了就是：

y = sign(w·x + b)

输入x乘以权重w，加上偏置b，过个阶跃函数就输出结果。简单吧？

但它有个致命问题——只能解决线性可分问题。我记得刚入行时，导师跟我说："你试试用单层感知机实现异或门。"我折腾了一整天，死活搞不定。后来才知道，Minsky早在1969年就证明了这一点。

核心要点：单层感知机 = 线性分类器。只能画一条直线把数据分开。

那怎么办？加层呗！

多层感知机（MLP）就是在输入和输出之间加了"隐藏层"。你想想看，一个隐藏层就能逼近任意连续函数了——这就是著名的万能逼近定理。

# 一个简单的MLP结构
输入层: 4个神经元
隐藏层: 8个神经元 (激活函数: ReLU)
输出层: 3个神经元 (激活函数: Softmax)

我的经验：隐藏层不是越多越好。我曾经在一个项目里堆了10层，结果训练慢得像蜗牛，还过拟合。后来砍到3层，效果反而更好。记住：深度够用就行，别贪多。

二、卷积神经网络(CNN)原理

CNN的出现，说白了就是解决了一个问题：全连接层参数太多了！

一张256x256的彩色图片，如果直接用全连接层，光输入层就有256*256*3=196,608个神经元。再连到隐藏层...参数数量直接爆炸。

CNN用了三个核心思想来降参数：

  • 局部连接：每个神经元只看一小块区域，不是整张图
  • 权值共享：同一个卷积核扫遍全图，参数共用
  • 池化：下采样，保留主要特征，丢掉冗余信息

我在做AI芯片时，最头疼的就是卷积运算的硬件加速。为什么？因为卷积本质上是滑动窗口的乘加运算，计算量巨大。一个3x3的卷积核，在256x256的特征图上滑动，要做(256-3+1)^2 * 9 ≈ 58万次乘加。这还只是一层！

避坑指南：我曾经设计过一款NPU，把卷积核设成了7x7，结果芯片面积超标30%。后来改成3x3堆叠，性能没降，面积省了一大截。记住：小卷积核堆叠比大卷积核更高效。

CNN的典型结构是这样的：

输入 → [卷积 → ReLU → 池化] × N → 全连接 → Softmax

其中N通常取2-5。VGGNet用了16层，ResNet做到了152层。但说实话，在NPU设计里，层数太多对硬件很不友好——片上存储根本放不下中间结果。

三、循环神经网络(RNN)与LSTM

CNN处理的是空间信息，那时间信息怎么办？比如语音、文本、股票价格——这些序列数据，前后是有依赖关系的。

RNN的思路很直接：把上一时刻的隐藏状态传到下一时刻。公式长这样：

h_t = tanh(W·[h_{t-1}, x_t] + b)

但RNN有个大坑——梯度消失/爆炸。序列一长（比如超过20步），前面的信息就传不过来了。我在做语音识别芯片时，试过用普通RNN处理10秒的语音，结果准确率惨不忍睹。

LSTM（长短期记忆网络）就是来解决这个问题的。它加了三个门：

  • 遗忘门：决定丢掉哪些旧信息
  • 输入门：决定记住哪些新信息
  • 输出门：决定输出什么

再加上一个细胞状态（cell state），像传送带一样贯穿整个序列。信息想传多远就传多远。

硬件视角：LSTM的计算量是普通RNN的4倍。每个时间步要算4组权重矩阵乘法。在NPU里，我通常会用专用的LSTM计算单元，把遗忘门、输入门、输出门和候选状态并行计算，这样能省不少时钟周期。

四、Transformer架构简介

2017年Google提出Transformer时，我第一反应是："又来一个新架构？"但仔细看完论文，我服了——这玩意儿确实牛。

Transformer的核心是自注意力机制。它不再像RNN那样一步步传状态，而是让序列里每个位置都能直接"看到"所有其他位置。

公式长这样：

Attention(Q, K, V) = softmax(Q·K^T / √d_k) · V

Q、K、V分别是查询、键、值矩阵。说白了就是：每个词去"问"其他词跟自己的相关性，然后根据相关性加权求和。

Transformer的优势很明显：

  • 并行计算：不像RNN必须串行，Transformer可以一次算完所有位置
  • 长距离依赖：任意两个位置的距离都是1步，不会信息丢失
  • 可扩展性：堆叠层数越多，效果越好（GPT-3用了96层）

我的建议：做NPU设计时，Transformer的注意力计算是最大的瓶颈。Q·K^T的矩阵乘法复杂度是O(n^2)，序列长度n一长（比如2048），计算量就爆炸。我通常会加一些稀疏化策略，或者用Flash Attention那样的分块计算，来降低显存带宽压力。

在NPU设计里，我个人的经验是：没有银弹。CNN适合图像，LSTM适合语音，Transformer适合文本。但现在的趋势是混合架构——比如MobileNet用CNN+Transformer，效果就很好。

总结一下：

  • 感知机：基础单元，线性分类
  • MLP：加隐藏层，万能逼近
  • CNN：局部连接+权值共享，适合图像
  • RNN/LSTM：处理序列，解决长距离依赖
  • Transformer：自注意力，并行计算，当前主流

这些基础概念搞清楚了，后面讲NPU架构设计时，你就能理解为什么某些硬件模块要那样设计。说白了，硬件是为算法服务的——算法怎么算，硬件就怎么加速。
