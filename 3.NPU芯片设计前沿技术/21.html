第21章：模拟与混合信号NPU

说实话，做数字芯片设计久了，总觉得模拟电路是个「玄学」。我早年做传统数字NPU时，一直觉得模拟计算是隔壁组的事。直到有一次，我参与了一个超低功耗边缘AI项目，功耗预算卡得死死的——数字方案怎么都压不下去。这才逼着我去认真研究模拟与混合信号NPU。嗯，这一研究，发现门道还真不少。

21.1 为什么需要模拟计算？

你想想看，传统数字NPU里，数据从存储到计算单元，再写回存储，每一步都在搬数据。搬数据这件事，功耗占比能到60%以上。模拟计算的核心思路是什么？说白了，就是直接在存储或信号域里完成计算，省掉搬来搬去的开销。

我在一个语音唤醒芯片项目中试过纯数字方案，功耗始终在毫瓦级别下不来。后来改用混合信号方案，把第一层特征提取用模拟电路做，功耗直接降了一个数量级。当然，代价是精度损失和设计复杂度上升。

核心优势：

  • 计算与存储融合，减少数据搬运
  • 利用物理定律（如基尔霍夫定律）直接完成乘加运算
  • 功耗可低至数字方案的1/10到1/100

21.2 模拟计算单元设计

模拟计算单元的核心是乘法累加（MAC）操作。数字MAC需要加法器和乘法器，而模拟MAC可以用一个电阻网络或电容阵列搞定。

21.2.1 基于电阻阵列的模拟MAC

最经典的做法是用电阻网络实现向量-矩阵乘法。输入电压代表激活值，电阻值代表权重，输出电流就是加权和。我当年第一次看到这个结构时，觉得太巧妙了——基尔霍夫电流定律直接帮你算完了。

// 模拟MAC的数学本质
// V_in: 输入电压向量 (N维)
// G: 电导矩阵 (N x M)
// I_out: 输出电流向量 (M维)

I_out[j] = Σ (V_in[i] * G[i][j])   // 基尔霍夫电流定律

// 权重更新：通过改变电阻值实现
// 实际实现中常用RRAM或浮栅晶体管

但这里有个坑。电阻阵列的精度受工艺偏差影响很大。我曾经在一个28nm工艺的项目里，电阻值偏差达到±20%，导致计算误差完全不可接受。后来我们用了差分结构，才把误差压到5%以内。

注意：模拟计算单元的精度受限于器件匹配度。RRAM（ resistive random-access memory）的电阻状态通常只有4-8个稳定级别，对应2-3比特精度。想做到8比特以上，需要复杂的校准电路。

21.2.2 基于电容的模拟计算

电容阵列是另一种常见方案。用电荷共享的方式实现加权求和。好处是功耗更低，因为电容不消耗静态电流。我建议在超低功耗场景优先考虑电容方案。

我记得有个项目做心电信号分类，输入信号本身只有8比特精度。用电容阵列做模拟计算，完全够用。功耗只有数字方案的1/20。

21.3 模数转换器（ADC）精度与功耗

模拟计算的结果最终要转成数字信号，才能做后续处理或存储。ADC是混合信号NPU的瓶颈——它决定了整个系统的精度和功耗上限。

21.3.1 ADC精度选择

精度每增加1比特，ADC功耗大约翻倍。所以不要盲目追求高精度。我见过一个团队，明明只需要4比特精度，非要用8比特ADC，结果功耗占了整个芯片的40%。

应用场景 推荐ADC精度 典型功耗（@1MS/s）
语音唤醒 4-6比特 10-50 μW
图像分类（边缘） 6-8比特 50-200 μW
高精度推理 8-10比特 200 μW - 1 mW
训练场景 10-12比特 1-5 mW

21.3.2 ADC架构选择

不同ADC架构的功耗-精度-速度特性差异很大：

  • 逐次逼近型（SAR ADC）：我最常用的方案。精度8-12比特，功耗适中，面积小。适合大多数NPU场景。
  • 闪存型（Flash ADC）：速度快，但精度低（通常6比特以下），功耗高。只在需要极高吞吐的场景用。
  • Σ-Δ型：精度极高（16比特以上），但速度慢。适合传感器信号处理，不适合实时推理。

我个人习惯在边缘NPU里用SAR ADC。它有个好处——可以随时关断，不工作时几乎不耗电。这在间歇性工作的IoT设备里特别实用。

21.4 混合信号训练与推理

混合信号NPU的训练是个大难题。为什么？因为模拟电路的梯度没法精确计算。

21.4.1 推理：相对简单

推理时，权重是固定的。模拟计算单元只需要做前向传播。误差主要来自ADC量化、器件噪声和温度漂移。这些可以通过校准和冗余设计来补偿。

我曾经做过一个实验：用4比特模拟计算做ResNet-18推理，在CIFAR-10上精度只比32比特浮点低了1.2%。嗯，这个结果让我对模拟推理有了信心。

21.4.2 训练：挑战巨大

训练需要反向传播，需要精确的梯度。模拟电路的梯度受噪声和器件非理想性影响，几乎没法直接使用。

目前主流做法是「数字训练，模拟推理」——在数字域用高精度完成训练，然后把量化后的权重映射到模拟计算单元。说白了，模拟只做推理，不做训练。

我的建议：如果你刚开始做混合信号NPU，先别碰模拟训练。把推理做好，用数字训练+模拟推理的方案，已经能覆盖90%的应用场景。模拟训练目前还在学术研究阶段，工程落地还需要时间。

21.5 噪声与误差容限

模拟电路最头疼的就是噪声。热噪声、闪烁噪声、电源噪声、衬底耦合噪声...种类多到让人崩溃。

21.5.1 噪声来源分析

  • 热噪声：所有电阻和晶体管都有。带宽越宽，噪声越大。我建议在模拟计算单元前加低通滤波，限制带宽。
  • 闪烁噪声（1/f噪声）：低频段主导。可以通过斩波技术或相关双采样来抑制。
  • 电源噪声：数字电路开关时产生的电源波动。需要独立的模拟电源域和良好的去耦设计。
  • 量化噪声：ADC量化过程引入的。精度每增加1比特，量化信噪比提升6 dB。

21.5.2 误差容限设计

好消息是，神经网络本身对噪声有一定容限。我做过测试：在推理时给权重加上5%的随机噪声，精度下降不到1%。这说明神经网络有天然的鲁棒性。

但容限不是无限的。以下情况需要特别注意：

  1. 关键层保护：第一层和最后一层对噪声最敏感。我建议在这些层使用更高精度的ADC。
  2. 冗余计算：对关键路径做双份或三份冗余，用多数表决来纠错。
  3. 自适应校准：定期用参考信号校准模拟计算单元，补偿温度漂移和老化效应。

避坑指南：我曾经在一个项目中忽略了温度漂移。芯片在25°C下测试完全正常，但到了65°C，精度直接掉了5%。后来加了片上温度传感器和动态补偿，才解决问题。记住：模拟电路的性能是温度的函数，一定要做全温度范围验证。
