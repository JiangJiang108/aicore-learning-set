第23章：光计算NPU——当光子取代电子

各位同学，今天我们来聊一个让我特别兴奋的话题——光计算NPU。说实话，我在这个领域摸爬滚打了快十年，从最初觉得“这玩意儿太科幻”到如今看到它逐步落地，心情还挺复杂的。

光计算，说白了就是用光子代替电子来做计算。你想想看，电子在导线里跑，有电阻、有电容，速度上不去，功耗还大。光子呢？在波导里几乎无损耗，速度是光速，带宽大得吓人。我2018年参与过一个光互联项目，当时就被光子的潜力震撼到了——一根光纤能顶上千根铜线的带宽。

23.1 光学矩阵乘法：光计算的核心算子

矩阵乘法是NPU的命根子。无论是卷积还是全连接，归根结底都是矩阵乘。光计算NPU的第一个突破口，就是光学矩阵乘法。

怎么用光做矩阵乘法？最经典的方法是“马赫-曾德尔干涉仪阵列”（MZI阵列）。每个MZI单元可以看作一个可调的光学“乘法器”，通过调整相位，实现权重矩阵的编码。输入光信号经过MZI网络，输出就是矩阵乘的结果。

核心原理：

  • 输入向量编码为光强或相位
  • 权重矩阵通过MZI阵列的相位调谐实现
  • 光信号在波导中传播，天然完成乘加运算
  • 输出端用光电探测器读出结果

我在项目中遇到过一个问题：MZI阵列的精度受限于相位调谐的准确性。硅基工艺下，热光调谐的精度大概在8-10位，远不如电子芯片的16位甚至32位。嗯，这里要注意——光计算目前更适合推理，训练的话精度还不够。

// 光学矩阵乘法的数学表示（伪代码）
// 输入向量 x，权重矩阵 W，输出 y = W * x
// 光学实现：每个MZI单元对应一个2x2酉矩阵
function optical_matmul(x, W):
    // 将W分解为MZI参数（相位θ, φ）
    params = decompose_to_mzi(W)
    // 配置MZI阵列
    configure_mzi_array(params)
    // 输入光信号
    optical_input = encode_to_optical(x)
    // 光信号传播，完成矩阵乘
    optical_output = propagate_through_mzi(optical_input)
    // 光电转换
    y = decode_from_optical(optical_output)
    return y

我个人习惯把光学矩阵乘法分成两类：相干光方案和非相干光方案。相干光用相位编码，精度高但系统复杂；非相干光用强度编码，简单但精度受限。你想想看，如果做边缘设备，非相干光可能更合适；如果是数据中心，相干光才是王道。

23.2 光子神经网络架构

有了光学矩阵乘法，我们就能搭建光子神经网络了。但别急着高兴——直接把电子神经网络搬到光域，效果往往不好。

为什么？因为光有光的脾气。比如非线性激活函数，电子芯片用ReLU、Sigmoid很简单，光域里怎么做？我见过有人用饱和吸收体做非线性，也有人用电光调制器做。说实话，目前还没有特别完美的方案。

常见的光子神经网络架构包括：

  • 全光神经网络（AONN）：所有层都用光学实现，包括非线性。优点是速度快，缺点是系统复杂，损耗大。
  • 光电混合神经网络：矩阵乘法用光，非线性用电子。这是目前最务实的方案，我参与的项目大多走这条路。
  • 脉冲光子神经网络：用光脉冲的时序编码信息，类似SNN。适合低功耗场景，但训练难度大。

避坑指南：我曾经在光子神经网络中忽略了“光损耗”问题。光在波导里传播，每经过一个MZI单元就有0.1-0.5dB的损耗。一个64x64的MZI阵列，光功率可能衰减到输入的1%以下。所以，光放大器是必须的，但放大器本身也会引入噪声。

我记得有一次做系统仿真，发现光信号经过20层MZI后，信噪比已经低到没法用了。后来我们加了掺铒光放大器（EDFA），但EDFA的增益平坦度又成了新问题。嗯，做光计算，系统思维很重要。

23.3 光电混合计算系统

光电混合，是目前最靠谱的路线。我甚至可以说，未来5-10年，纯光NPU不太可能商用，但光电混合NPU已经在路上了。

一个典型的光电混合计算系统长这样：

  • 光域部分：做矩阵乘法、卷积等计算密集型操作
  • 电域部分：做非线性激活、控制逻辑、数据缓存
  • 光-电接口：高速光电探测器（PD）和调制器，负责信号转换

我参与过一个项目，目标是做100TOPS/W的光电混合NPU。电子NPU的能效比大概在10-20TOPS/W，光计算理论上能到1000TOPS/W以上。但实际做下来，光-电转换的功耗占了总功耗的30%以上。说白了，接口才是瓶颈。

警告：光电混合系统的“延迟匹配”是个大坑。光信号传播速度是3×10^8 m/s，电子信号在PCB上只有0.5-0.7倍光速。如果光路和电路的长度不匹配，时序就会乱。我曾经因为没注意这个，导致整个系统跑不起来，排查了整整两周。

23.4 光计算芯片制造挑战

说到制造，这才是光计算NPU真正头疼的地方。电子芯片的制造工艺已经成熟到3nm、2nm，光计算芯片呢？还在襁褓里。

主要挑战有这几个：

挑战 具体问题 我的经验
工艺兼容性 硅光工艺与CMOS工艺不兼容，需要额外步骤 我曾经用180nm CMOS做硅光，良率只有60%
光损耗 波导、分束器、MZI都有损耗，累积效应严重 一个64x64 MZI阵列，光功率衰减到1%以下
热稳定性 硅的折射率随温度变化，相位调谐会漂移 温度每变化1°C，相位漂移约0.1π
封装 光纤耦合对准精度要求亚微米级 封装成本占整个芯片成本的40%以上
测试 光芯片测试需要专用设备，效率低 一片晶圆测试要3天，电子芯片只要3小时

重要提醒：光计算芯片的“热管理”比电子芯片更棘手。电子芯片发热，你可以用散热片、液冷。光芯片呢？温度变化直接影响相位，导致计算错误。我曾经在项目中用TEC（热电制冷器）控温，但TEC本身功耗很大，把光计算的能效优势全抵消了。

还有一个容易被忽视的问题——光源集成。目前大多数光计算芯片用外部激光器，通过光纤耦合。但外部激光器体积大、成本高。片上激光器（比如量子点激光器）是方向，但工艺还不成熟。我记得2022年看过一篇论文，片上激光器的寿命只有几千小时，远不如商用激光器的10万小时。

嗯，说了这么多挑战，是不是觉得光计算NPU前途渺茫？其实不然。我个人的判断是：光计算不会取代电子计算，但会在特定场景（比如数据中心、高性能计算、AI推理）中发挥独特优势。就像GPU没有取代CPU，而是共存一样。

最后，我想分享一个观点：做光计算芯片，不要追求“全光”，而要追求“好用”。光电混合，取长补短，才是务实的路线。我见过太多团队执着于纯光方案，结果项目流产。反倒是那些务实的光电混合方案，已经流片成功，开始小批量出货了。

我的建议：如果你刚接触光计算，先从光电混合系统入手。用电子芯片做控制和非线性，用光芯片做矩阵乘法。这样既能体验光计算的优势，又能规避制造和封装的坑。等积累了经验，再考虑更激进的方案。

好了，这一章的内容就到这里。光计算NPU是个快速发展的领域，每半年就有新突破。保持关注，保持学习，说不定下一个改变行业的人就是你。
